# Section 4.12: Workspace & Collaboration - Implementation Plan

## Overview
Build polymorphic commenting system, @mentions, teams, file attachments, e-signature generalization, discussion boards, notification centre, and shared calendar.

## Existing Foundation
- `DealComment` + `DealCommentMention` - deal-specific comments
- `TicketInternalNote` + `InternalNoteMention` - ticket-specific notes
- `Team` + `TeamMember` - basic team tables (needs `is_archived`, `team_lead_id`, `is_primary`)
- `ContractSignatory` - e-signature infrastructure exists
- `spatie/laravel-medialibrary` installed for file attachments
- `InternalNoteMentioned` notification exists for mentions
- Teams already used in `TicketCategory` for routing

- Create `HasAttachments` trait with file validation
- Generalize e-signature for any attachment (Signable trait)

## Key Decisions
1. Comments use `HasUlids` - polymorphic tables with ULID foreign keys
2. 15-minute edit window for comments (standardized across all entities)
3. Soft-delete for comments via `deleted_at` placeholder
4. Calendar colours match contract status badge colours (draft=blue, sent=indigo, signed=teal, active=green, expiring=amber, expired=red)
