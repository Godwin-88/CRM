Based on section 4.4 of the README — **Marketing & Campaign Orchestration** — here are the user stories and acceptance criteria for each listed feature.

---

## Feature 1: Campaign builder with drag-and-drop email template editor

**Story:** As a marketing manager, I can build email campaigns using a visual drag-and-drop editor so that I can create professional campaign content without writing HTML.

Acceptance criteria:
- The editor provides a canvas where the agent drags content blocks from a sidebar panel. Available block types are: header, text, image, button, divider, spacer, and social links.
- Blocks can be reordered by dragging within the canvas. A block can be duplicated or deleted via a context menu on hover.
- Each block has configurable properties accessible in a right-hand settings panel when the block is selected. Text blocks support bold, italic, link, and bullet list formatting.
- The editor renders a live preview of the email as it is built. A mobile preview toggle shows how the template renders at 375px width.
- Variable placeholders are insertable from a dropdown within any text block. Available variables include contact first name, account name, agent name, unsubscribe link, and any active custom contact fields.
- Templates are saveable and reusable across campaigns. Saving a template does not publish or schedule the campaign.
- The editor exports a clean HTML email compatible with major email clients (Gmail, Outlook, Apple Mail). The exported HTML is stored alongside the template record.
- An agent can switch to a raw HTML edit mode for advanced customisation. Switching back to visual mode is blocked if the HTML contains structures the visual editor cannot represent, with a clear warning shown.

## Feature 2: Multi-channel campaigns / social channels(WHatsapp, facebook, instagra, tiktok , linkedin)

**Story:** As a marketing manager, I can create campaigns that send across email, SMS, push, and in-app channels in a defined sequence so that I reach contacts through multiple touchpoints in a coordinated way.

Acceptance criteria:
- A campaign can include one or more steps. Each step specifies a channel (email, SMS, push, or in-app), a template, and a delay from the previous step (immediately, after N hours, after N days).
- The first step can have no delay. Subsequent steps must have a delay of at least 1 hour.
- Each step references a separately configured template appropriate to its channel — email templates from the email builder, SMS templates from the SMS template library, push and in-app templates from their respective editors.
- When a campaign is dispatched, step one is sent to all contacts in the target segment. Subsequent steps are queued as delayed jobs, one job per contact per step.
- A contact who unsubscribes or opts out after step one is excluded from all remaining steps automatically. The exclusion is applied at job execution time, not at schedule time.
- The campaign detail view shows delivery progress per step — total sent, delivered, failed, and pending counts updating as the campaign runs.
- A multi-channel campaign can be paused by a manager after dispatch. Pausing prevents pending step jobs from executing. Resuming re-queues them.
- Channel availability per step is constrained by the contact's channel preferences. A step targeting SMS is skipped for contacts with SMS opted out, and the next step executes on schedule.

## Feature 3: Segment targeting with real-time count preview

**Story:** As a marketing manager, I can select a segment as the target audience for a campaign and see the live contact count before sending so that I know exactly how many contacts will receive it.

Acceptance criteria:
- During campaign setup, the manager selects one segment as the target audience from the existing segment library or creates a new one inline.
- On segment selection, the system immediately queries the current matching contact count and displays it alongside the segment name.
- The count reflects only contacts who are active (not soft-deleted, not churned unless the segment explicitly includes churned contacts) and have not globally opted out.
- A breakdown of the count by channel is shown — e.g. "820 email-eligible, 640 SMS-eligible, 310 push-eligible" — so the manager understands effective reach per channel.
- A sample of up to 10 matching contacts is displayed so the manager can sanity-check the audience.
- If the segment count is zero, the campaign cannot be advanced to the scheduling step. A validation message explains the block.
- The count is re-evaluated when the campaign moves from draft to scheduled. If the count has changed significantly (more than 20% variance from the draft count), a warning is shown requiring confirmation before proceeding.
- The count preview does not persist — it is a live query each time it is requested and is not cached beyond the current session.

## Feature 4: A/B testing

**Story:** As a marketing manager, I can set up A/B tests within a campaign so that I can identify which variant performs better and optimise future sends.

Acceptance criteria:
- A/B testing is available for email campaigns. Testable dimensions are: subject line, email content variant, and send time.
- A campaign can have a maximum of two variants (A and B) per test. Testing more than two variants simultaneously is not supported.
- The manager specifies the percentage of the target segment to include in the test. The remainder is held as the control group and receives the winning variant after the test concludes.
- Test duration is configurable: the winning variant is determined after a set number of hours (minimum 1, maximum 72) or when a statistical threshold is met based on open rate.
- The winning criterion is configurable: highest open rate, highest click rate, or highest conversion (click-through to a tracked URL).
- During the test period, the campaign detail view shows live metrics for each variant side by side.
- At test conclusion, the system identifies the winning variant and automatically sends it to the control group. The manager receives an in-app notification with the test outcome summary.
- If neither variant achieves a statistically meaningful difference after the test duration, the system selects variant A as the winner by default and notifies the manager of the inconclusive result.
- A/B test results are stored on the campaign record and included in campaign analytics exports.

## Feature 5: Campaign schedule and throttle control

**Story:** As a marketing manager, I can schedule a campaign for a future date and time and configure a send rate throttle so that sends are controlled and do not overwhelm delivery infrastructure or contact inboxes.

Acceptance criteria:
- A campaign can be scheduled for an immediate send or a future date and time. The scheduled time is stored in UTC and displayed in the manager's local timezone.
- Scheduling requires the campaign to be in a valid state — target segment selected, at least one step with a template assigned, and the campaign passing all pre-send validation checks.
- Pre-send validation checks include: segment count greater than zero, all templates passing spam score check, unsubscribe link present in all email templates, and sender address verified with the mail provider.
- A throttle rate can be configured as a maximum number of sends per hour per channel. Default throttle is 5,000 emails per hour and 1,000 SMS per hour unless overridden by an admin.
- The dispatch job respects the throttle by chunking the contact list and introducing queue delays between chunks. Chunk size is derived from the throttle rate and the queue worker concurrency setting.
- A scheduled campaign can be cancelled up to 5 minutes before its scheduled time. After that window, cancellation is only available while the send is in progress, and only unsent records are cancelled — already-sent records are not recalled.
- The campaign detail view shows a countdown timer to the scheduled send time while the campaign is in scheduled status.
- Send time optimisation (sending to each contact at the time of day they are most likely to engage) is a configurable option per campaign. When enabled, the scheduled time becomes the earliest send time and individual sends are distributed across a 24-hour window.

## Feature 6: Automated drip sequences

**Story:** As a marketing manager, I can build automated drip workflows triggered by contact events so that contacts receive timely, relevant communications without manual intervention.

Acceptance criteria:
- A drip sequence is defined by a trigger, one or more conditions, and an ordered list of actions.
- Supported triggers are: contact created, contact stage changed, deal stage changed, contact field value changed, form submission, and manual enrolment.
- Supported conditions are: contact type equals, segment membership, custom field value, and time since trigger (wait N hours/days before next step).
- Supported actions are: send email, send SMS, send in-app notification, create activity, update contact field, add to segment, remove from segment, and notify agent.
- A contact can only be enrolled in a given drip sequence once at a time. Re-enrolment after completion is configurable per sequence.
- A contact who meets an exit condition (configurable, e.g. deal closed won, opted out) is removed from the sequence immediately and their pending steps are cancelled.
- Sequence execution is fully queue-driven. Each step for each contact is a separate queued job. Failed jobs retry three times before marking the contact's sequence enrolment as errored.
- The sequence detail view shows active enrolments, completed enrolments, and errored enrolments with the ability to drill into individual contact progress.
- Drip sequences can be activated and deactivated. Deactivating a sequence pauses all pending steps for currently enrolled contacts.
- Building and activating sequences is restricted to manager role and above.

## Feature 7: Unified tag management and UTM tracking

**Story:** As a marketing manager, I can tag campaigns with UTM parameters and internal labels so that campaign traffic is attributable in analytics and campaigns are organised consistently.

Acceptance criteria:
- Every campaign has an optional internal tag field supporting multiple free-text tags. Tags are autocompleted from existing tags as the manager types.
- UTM parameters (source, medium, campaign, term, content) are configurable per campaign. When set, all links in email templates are automatically appended with the UTM parameters on send. The agent does not need to manually add them to each link.
- UTM parameters are validated to contain no spaces or special characters that would break URL structure. Invalid values show an inline error before the campaign can be saved.
- Click-through links in emails are routed through a CRM tracking redirect URL before forwarding to the destination. This captures the click event against the contact and the campaign for analytics.
- Tags are searchable in the campaign list view. Filtering the list by one or more tags returns only matching campaigns.
- Tags can be bulk-applied or removed from multiple campaigns simultaneously from the campaign list view.
- UTM tracking links and click events feed into the campaign analytics dashboard under the click metrics section.
- A contact's click on a tracked link is recorded as an interaction event of type email click against their contact record.

## Feature 8: Campaign analytics

**Story:** As a marketing manager, I can view detailed performance analytics for each campaign so that I can measure effectiveness and report on ROI.

Acceptance criteria:
- The campaign analytics view shows: total sent, delivered, bounced, open rate, click rate, unsubscribe rate, conversion count, and revenue attributed.
- Metrics are shown as absolute counts and percentages. Open rate and click rate are calculated on delivered count, not total sent.
- A time-series chart shows opens and clicks over time from send date, bucketed by hour for the first 48 hours and by day thereafter.
- Per-link click breakdown shows each tracked URL in the campaign, its click count, and unique click count.
- Per-contact engagement is accessible from the analytics view — a list of contacts who opened, clicked, converted, or unsubscribed, filterable by engagement type.
- Revenue attributed is calculated from deals that moved to Closed Won within 30 days of a contact clicking a tracked campaign link. The attribution window is configurable per campaign.
- Bounce handling: hard bounces automatically mark the contact's email as invalid and suppress them from future email sends. Soft bounces are recorded but do not suppress.
- Analytics data is available for export as CSV from the campaign analytics view.
- Cross-campaign analytics (aggregate metrics across multiple campaigns filtered by tag, channel, date range, or segment) are available in the analytics module under the MarCom dashboard.

## Feature 9: Mobile marketing and social media scheduling

**Story:** As a marketing manager, I can schedule posts to social media channels from within the CRM so that organic social activity is coordinated with campaign sends.

Acceptance criteria:
- Supported social channels for post scheduling are LinkedIn, X (Twitter), and Facebook. Each channel requires an authorised connection via OAuth in the integrations settings panel.
- A post is composed with a text body and an optional image attachment. Character limits per channel are enforced in the editor in real time (LinkedIn 3,000 chars, X 280 chars, Facebook 63,206 chars).
- A post can target one or more connected channels simultaneously. Each channel receives the same content unless the manager toggles per-channel editing, in which case each channel's content is edited independently.
- Posts are scheduled for a specific date and time. The scheduler dispatches the post via each channel's API at the scheduled time as a queued job.
- Published posts show their engagement metrics (likes, comments, shares, impressions) pulled from the channel API on a 6-hour refresh cycle.
- A post can be in draft, scheduled, published, or failed status. A failed post surfaces an error reason from the channel API and allows the manager to retry.
- Social post scheduling is available to manager role and above. Agents cannot create or schedule posts.
- Social posts are linked to a campaign optionally. When linked, the post appears in the campaign's activity timeline for a unified view of all campaign touchpoints.

## Feature 10: Content lifecycle management

**Story:** As a marketing manager, I can manage marketing content through a defined lifecycle from draft to archive so that only approved content is used in campaigns and outdated content is retired cleanly.

Acceptance criteria:
- Content items (email templates, SMS templates, landing page copy, social post templates) have a status of draft, in review, approved, published, or archived.
- A content item in draft can be edited freely. Once submitted for review it is locked for editing by the original author — only a reviewer can make changes or send it back to draft.
- The review workflow requires at least one approver. The approver is assigned per content type in system settings. The approver receives an in-app and email notification when a content item is submitted for review.
- Only approved content can be selected in a campaign builder or drip sequence. Draft and in-review items do not appear in template selectors.
- Publishing an approved item makes it the active version. A previously published version of the same template is automatically archived when a new version is published.
- Archiving a content item removes it from template selectors in new campaigns. Existing campaigns already using the archived template are not affected — they retain a reference to the archived version.
- All status transitions are logged in the audit trail with the acting user, timestamp, and previous status.
- Content items are versioned. Each save of a draft creates a new version record. An admin can restore a previous version, which creates a new draft from the restored content.
